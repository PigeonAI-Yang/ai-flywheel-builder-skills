# 默认 HTML 报告主题

## 定位

这是 AI Flywheel Builder 的开源默认主题。

它保证没有任何私有设计 skill、品牌资产或外部依赖时，系统仍然能生成可读、可发布、可归档的单文件 HTML 主交付物。

## 使用场景

- 用户没有指定设计主题。
- 用户没有安装本地设计 skill。
- 开源版本在新机器上首次运行。
- 内容需要先保证清晰可读，而不是追求品牌化视觉。

## 输出要求

生成 `content-flywheel-run.html` 时，默认主题必须满足：

- 单文件 HTML，可直接用浏览器打开。
- 内联 CSS，不依赖远程字体、图片或脚本。
- 中文优先，按用户语言生成界面文案。
- 明确显示本轮实践、内容草稿、领取产物、证据边界、反馈入口和下一轮实践。
- 移动端可读，正文不溢出。
- 不显示后台运行信息。run、索引、验收、旧输出作废、后台记录和已更新文件只进入后台文件或最终回复路径清单。

## 视觉原则

- 白底或浅色底，正文高对比。
- 标题、正文、证据、风险和下一步动作有清晰层级。
- 不使用过度装饰、复杂动画或平台风格强绑定元素。
- 不把内部验收、脚本输出、索引路径写成对外正文主线。

## 最小结构

```text
封面
本轮实践一句话
为什么值得讲
信息增量
内容框架稿
可发布内容草稿
配套可领取产物
证据与边界
反馈入口
下一轮实践
```
